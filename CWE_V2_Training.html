System Overview

What is the Claims Workflow Engine?

A Google Sheets-based claims management system that guides billing staff through every step of claim resolution.

🎯 Purpose

The Claims Workflow Engine V2.0 (CWE) standardizes how your organization handles denied, pended, and underpaid claims. It replaces ad-hoc tracking with a structured, auditable process that every team member follows the same way.

🗂️ Core Components

  • Claims Sheet — Main working area; one row per claim
  • Sidebar Panel — Context-sensitive guidance for the selected claim
  • REF-CARC Sheet — Lookup table for denial codes and actions
  • REF-MACs Sheet — State-to-MAC mapping for Medicare claims
  • REF-Payers Sheet — Payer-specific rules and contacts
  • Workflow Guide — Decision trees embedded in the sidebar

🔄 Claim Lifecycle

Every claim moves through defined stages. The system enforces valid transitions and logs timestamps at each stage change.

NEW → WORKING → ESCALATED → RESOLVED or CLOSED

Claims can also move to PENDING INFO when waiting on documentation, or APPEALED when a formal appeal is submitted.

✨ New in V2.0

  • MAC Lookup — Medicare claims now auto-display the Part B and DME MAC for the practice state, so billers know exactly who to call
  • CARC Actions — Each denial code now surfaces a specific recommended action in the sidebar header — no more hunting through policy docs
  • Improved Escalation Routing — Escalated claims auto-assign to supervisor queue with aging alerts
User Manual

Navigating the System

How to open and move around the Claims Workflow Engine in Google Sheets.

Opening the System

Open the Google Sheet
Navigate to the shared "Claims Workflow Engine V2.0" spreadsheet in Google Drive. Bookmark it for easy access.
Open the Sidebar
Go to Extensions → Claims Workflow → Open Sidebar. The sidebar will open on the right side of your screen.
Select a Claim Row
Click any cell in a claim row. The sidebar automatically loads information for that claim, including CARC action guidance and (for Medicare) MAC info.
Work the Claim
Follow the guidance in the sidebar. Use the action buttons to advance the stage, add notes, or escalate.

Using the Sidebar

The sidebar is your control center for each claim. It surfaces everything you need in one place.

Sidebar Sections

📌 Claim Header

The top section shows key claim identifiers: Claim ID, Patient Name, DOS, Payer, and current Stage. The stage badge updates in real time as the claim progresses.

⚡ CARC Action Banner NEW V2.0

When a claim has a CARC (Claim Adjustment Reason Code), the sidebar now shows three pieces of information:

  • CARC Code — The numeric denial reason (e.g. CO-16)
  • CARC Description — Plain English explanation of the denial
  • ⚡ Action — A specific recommended next step, pulled directly from your REF-CARC reference table. This appears as an orange highlighted banner between the description and the Stage field.
✅
If no CARC action is configured for a code, the Action field is hidden — it only appears when actionable guidance exists.

🏥 MAC Information NEW V2.0

For Medicare claims only, the sidebar now shows a "Medicare Administrative Contractors" section displaying the Part B MAC and DME MAC for the practice's state. This section is hidden for all non-Medicare payers.

Use the MAC name and jurisdiction code when calling for claim status or submitting reopening requests.

📝 Notes & Actions

The bottom of the sidebar contains:

  • Notes field — Timestamped audit trail for all actions taken
  • Stage Advance buttons — Move the claim forward (only valid next stages are shown)
  • Escalate button — Routes to supervisor queue with required note prompt
  • Close button — Marks as unworkable with required reason code
User Manual — New Feature

MAC Lookup

How the system finds your Medicare Administrative Contractor information automatically.

🔍
MAC information only appears when the Payer Type for a claim is set to Medicare. For all other payers, this section stays hidden.

What is a MAC?

Medicare Administrative Contractors (MACs) are private companies that Medicare has paid to process Medicare claims. There are two types relevant to billing:

  • Part B MAC — Processes outpatient professional services (physician and supplier claims). Contact for claim status, reopenings, redeterminations.
  • DME MAC — Processes durable medical equipment, prosthetics, orthotics, and supplies. Only relevant for DME claims.

Each MAC covers specific states and is identified by a Jurisdiction code (e.g., Jurisdiction J — Palmetto GBA covers GA, KY, NC, SC, VA, WV).

How the Lookup Works

Payer type is checked
When you select a claim row, the sidebar reads the Payer Type field. If it's not "Medicare," the MAC section stays hidden.
Practice state is read
The system reads the practice state from the claim record (2-letter abbreviation).
REF-MACs table is queried
The getMACsForState() function scans the REF-MACs reference sheet for a matching state and returns the Part B and DME MAC names and jurisdiction codes.
Results display in sidebar
The MAC section appears with both MAC names and their jurisdiction codes, formatted as cards for quick scanning.

CARC Actions

How the system guides you to the right next step for each denial reason code.

📖 What is a CARC?

A Claim Adjustment Reason Code (CARC) is a standardized code that payers attach to remittance advice to explain why a claim was adjusted or denied. Examples:

CodeDescription
CO-4The procedure code is inconsistent with the modifier
CO-16Claim/service lacks information which is needed for adjudication
CO-18Exact duplicate claim/service
CO-45Charges exceed your contracted/legislated fee arrangement
CO-97The benefit for this service is included in the payment/allowance for another service/procedure
PR-1Deductible amount
PR-2Coinsurance amount
PR-3Co-payment amount

How CARC Actions Work in V2.0

Payer posts an EOB/ERA
When a claim is denied or adjusted, enter the CARC code from the remittance into the CARC field on the claim row.
System looks up the code
The getIssueData() function calls getCARCAction() which scans Column E of the REF-CARC sheet for the suggested action for that specific code.
Action displays in sidebar
The orange "⚡ Action" banner appears between the CARC Description and the Stage field. It tells you exactly what to do — no policy manual needed.
You take the action
Follow the guidance, document your action in the Notes field, then advance the stage or escalate as appropriate.
⚠️
CARC actions in the REF-CARC sheet are maintained by your supervisor. If you believe an action is wrong or missing for a code, notify your supervisor — do not edit the REF-CARC sheet yourself.

Workflow Stages

Understanding the lifecycle of a claim and valid stage transitions.

StageMeaningWho Sets ItNext Valid Stages
NEW Claim just entered system, not yet reviewed System (auto) WORKING, CLOSED
WORKING Biller is actively working the claim Biller PENDING INFO, APPEALED, ESCALATED, RESOLVED, CLOSED
PENDING INFO Waiting on documentation or info from patient/provider Biller WORKING, CLOSED
APPEALED Formal appeal submitted Biller / Supervisor WORKING, RESOLVED, CLOSED
ESCALATED Needs supervisor review Biller WORKING, APPEALED, RESOLVED, CLOSED
RESOLVED Claim paid correctly; no further action needed Biller / Supervisor (terminal — reopen with WORKING if needed)
CLOSED Claim written off or deemed unworkable Supervisor approval required (terminal)
ℹ️
The system only shows buttons for valid next stages based on the claim's current stage. You cannot skip stages or go backward without supervisor override.

Stage Change Rules

  • Every stage change is timestamped and logged with the user's email
  • Moving to ESCALATED requires a note explaining why
  • Moving to CLOSED requires supervisor approval and a close reason code
  • Moving to RESOLVED requires a payment amount entry
  • Claims open longer than 30 days trigger an aging alert in the supervisor dashboard
Role Tutorial

Biller Workflow

👤 Biller   Your daily process for working the claims queue.

Daily Startup

Filter to your assigned claims
Use the filter on Column "Assigned To" to see only your claims. Sort by Days Open (descending) to prioritize oldest claims first.
Prioritize by urgency
Work claims in this order: (1) ESCALATED responses due, (2) PENDING INFO follow-ups, (3) NEW claims within timely filing window, (4) WORKING claims aged 15+ days.
Open the sidebar
Extensions → Claims Workflow → Open Sidebar. Keep it open all day.

Working a Denied Claim

Select the claim row
The sidebar loads automatically. Review the Claim Header for payer, DOS, and billed amount.
Enter the CARC code from the EOB
Type the CARC code into the designated column. The sidebar immediately shows the code description and the recommended action.
Read the ⚡ Action guidance
The orange action banner tells you exactly what to do. Follow it. If the guidance says "Resubmit with corrected modifier," do that. If it says "Obtain patient authorization on file," request that.
For Medicare: check MAC info
If the claim is Medicare, the MAC section shows your Part B MAC name and jurisdiction. Call that MAC's provider line for real-time claim status.
Document your action in Notes
Always enter a note before advancing the stage: what you did, who you spoke to, reference/confirmation numbers.
Advance the stage or escalate
Use the stage buttons in the sidebar. If you cannot resolve the claim yourself, use "Escalate" and describe why in the prompt.
📋
Timely filing rule: Know your payers' filing deadlines. Most commercial payers allow 90–180 days from DOS. Medicare allows 1 year. Flag claims within 30 days of their deadline as HIGH PRIORITY immediately.

When to Escalate

  • Claim involves a charge over $500 and has been denied twice
  • CARC action guidance says "Supervisor review required"
  • You believe the payer made an error but need authority to appeal
  • Claim is within 15 days of timely filing deadline and not yet resolved
  • Patient has contacted the office about the claim
Role Tutorial

Supervisor Workflow

👑 Supervisor   Managing the queue, handling escalations, and maintaining reference data.

Daily Supervisor Tasks

Review the REPORTS tab
Check the daily dashboard: escalated claim count, aging summary, top denial codes, and team productivity.
Work the ESCALATED queue
Filter the CLAIMS sheet to Stage = ESCALATED. Review each claim's note history. Take action or reassign within 24 hours of escalation.
Review aging alerts
Claims aged 30+ days appear highlighted. Verify they are actively being worked and have notes. Reassign if stale.

Maintaining Reference Data

Updating CARC Actions (REF-CARC Column E)

When you identify that a CARC action needs updating:

  • Go to the REF-CARC sheet
  • Find the row for the CARC code (Column A)
  • Edit Column E with the updated suggested action
  • The change takes effect immediately for all new sidebar loads
  • Document the change in the "Notes" column (if present) with date and your initials

Updating MAC Data (REF-MACs)

CMS periodically reassigns MAC jurisdictions. When this happens:

  • Check cms.gov for the updated MAC assignment list
  • Update the REF-MACs sheet: Column C (Part B Name), D (Part B J-Code), E (DME Name), F (DME J-Code)
  • Test by selecting a Medicare claim from an affected state and verifying the sidebar shows the correct MAC

Closing Claims

Only supervisors can move claims to CLOSED. When a biller requests a close:

  • Review the note history to confirm sufficient work was done
  • Select a Close Reason Code from the dropdown
  • Add a supervisor note with your decision
  • Advance to CLOSED — this action cannot be reversed without contacting IT

Trend Analysis

📊
Use the REPORTS tab weekly to identify top CARC codes by volume and dollar amount. If CO-16 ("missing information") is your #1 denial, that's a front-end data entry problem. Address root causes, not just individual claims.

Scenario: Commercial Claim Denied CO-4 (Modifier Mismatch)

Claim ID:CL-2024-00447
Patient:Johnson, Patricia
DOS:2024-11-14
Payer:Aetna
CPT:99214-25
Billed:$275.00
CARC:CO-4
⚡ Action:Verify modifier with provider. If -25 appropriate, resubmit with operative note. If not, resubmit without modifier.
Stage:NEW
Advance to WORKING
Click "Start Working" in the sidebar. Add note: "CO-4 denial received. Reviewing modifier usage."
Verify with provider
Contact the billing provider or pull the original encounter note. Confirm whether the -25 modifier (significant separate E&M same day as procedure) was correctly appended.
Correct and resubmit
If modifier was correct: resubmit with the operative note attached per Aetna's requirements. If modifier was incorrect: resubmit CPT 99214 without the -25 modifier.
Document and advance to APPEALED
Note: "Resubmitted 2024-11-19 with corrected claim. Ref# 447892. Modifier -25 confirmed appropriate per attached note." Advance stage to APPEALED.
Follow up and resolve
When payment posts, advance to RESOLVED. Enter payment amount. Note: "Paid $198.00 on 2024-12-02 per ERA 44871."

Scenario: Formal Appeal — Medical Necessity Denial

Claim ID:CL-2024-00512
Patient:Martinez, Robert
Payer:UnitedHealthcare
Billed:$1,450.00
CARC:CO-50
⚡ Action:Prepare medical necessity appeal. Obtain clinical notes, diagnostic results. Submit via payer appeal portal within 120 days of denial.
Stage:ESCALATED
Supervisor reviews escalation
Supervisor opens the claim. Reviews biller's notes. Confirms medical necessity appeal is appropriate. Moves back to WORKING and assigns to senior biller.
Gather clinical documentation
Request: encounter notes, lab/imaging results, prior authorization records, referring physician letter of medical necessity if available.
Draft and submit appeal
Submit via UHC's online appeal portal. Attach all clinical documentation. Include appeal letter citing UHC medical policy and clinical evidence.
Advance to APPEALED, set follow-up
Note full submission details. Stage → APPEALED. Set a calendar reminder to follow up in 30 days (or per UHC's stated turnaround).

Scenario: Medicare CO-16 — Missing Information

Claim ID:CL-2024-00389
Payer:Medicare
State:FL
Part B MAC:First Coast Service Options — Jurisdiction N
DME MAC:CGS Administrators — Jurisdiction C
CARC:CO-16
⚡ Action:Check ANSI 277 for specific missing element. Common issues: NPI mismatch, missing taxonomy code, invalid diagnosis pointer. Correct and resubmit.
Note MAC from sidebar
The sidebar shows the MAC for FL is First Coast Service Options (Jurisdiction N). Have the MAC provider line number ready before calling.
Call MAC for specific reason
CO-16 is a catch-all — call First Coast to get the specific reason segment (often AAR) that tells you exactly what's missing.
Correct the identified element
In your practice management system, fix the specific field identified (e.g., add missing taxonomy code, correct referring NPI).
Resubmit as corrected claim
Submit using bill type "7" (corrected claim) if applicable. Note the correction and confirmation number.

Scenario: Timely Filing Write-Off

Claim ID:CL-2023-00118
DOS:2023-01-15 (>365 days)
Payer:Medicare
CARC:CO-29 (Timely Filing)
⚡ Action:Obtain proof of timely filing (clearinghouse acceptance report). If unavailable, initiate write-off process per supervisor protocol.
Stage:ESCALATED
Attempt to locate proof of timely filing
Check clearinghouse reports, 277 acceptance reports, and any prior submission history. If claim was originally submitted on time but rejected, you may have grounds to appeal.
If no proof exists — escalate for write-off
Bring to supervisor with full note history. Supervisor reviews and determines if any exception applies.
Supervisor approves write-off
Supervisor selects Close Reason: "TF — Timely Filing Expired." Advances to CLOSED. This populates the write-off tracking report.
Root cause documentation
Supervisor documents root cause (front desk didn't verify insurance at time of service, etc.) to prevent recurrence. This feeds the monthly denial analysis report.
